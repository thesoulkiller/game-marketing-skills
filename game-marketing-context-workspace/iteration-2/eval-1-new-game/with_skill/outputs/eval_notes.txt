EVAL NOTES — game-marketing-context skill
Eval: iteration-2 / eval-1-new-game / with_skill
Eval date: 2026-04-30
Game: Pre-launch fantasy city-builder puzzle (F2P, age 18-35)
Skill version tested: 1.1.0

=== ITERATION 2 SKILL IMPROVEMENTS OVER ITERATION 1 ===

Iteration 1 eval notes identified 8 gaps. Here is how the iteration-2 skill
(v1.1.0) addressed each one:

1. NO WORKING TITLE GUIDANCE [iteration-1 gap]
   STATUS: NOT ADDRESSED in skill v1.1.0
   The skill still instructs to capture "Full title" without guidance for pre-launch
   games without a finalized name. This eval again had to note this gap explicitly in
   the output document and suggest using a working title. The skill should add a note:
   "If the title isn't finalized, use a working title and mark it as such."

2. COMPETITIVE LANDSCAPE SCAFFOLDING [iteration-1 gap]
   STATUS: NOT ADDRESSED
   The skill still asks for top 2–3 competitors without providing any research guidance
   (e.g., "check App Store Top Charts > Games > Strategy for direct comps"). In this
   eval, competitors were supplied via domain knowledge. Users without mobile gaming
   experience would struggle here.

3. UA CHANNEL LIST ACCURACY [iteration-1 gap]
   STATUS: FULLY ADDRESSED — significant improvement
   The skill v1.1.0 now explicitly uses current network names:
   - "Unity LevelPlay (formerly IronSource — merged 2023)"
   - "Liftoff (formerly Vungle — rebranded 2023)"
   - Added Moloco and Mintegral
   This directly resulted in a more accurate UA channel section in the output
   document. The iteration-1 output incorrectly listed "IronSource" and "Vungle"
   as separate current entities.

4. SOFT LAUNCH MARKET RATIONALE [iteration-1 gap]
   STATUS: FULLY ADDRESSED — major improvement
   The skill v1.1.0 now includes a complete "Soft launch market selection" explanation
   with five specific reasons (English-speaking, similar CPI to US, iOS-heavy, outside
   GDPR, small enough). This directly shaped the output document's Launch Status
   section — which now explains the "why" of CA/AU/NZ selection, not just lists them.
   This is one of the most meaningful skill improvements and was visibly reflected in
   the output quality.

5. CORE LOOP ARTICULATION TEMPLATE [iteration-1 gap]
   STATUS: FULLY ADDRESSED — significant improvement
   The skill v1.1.0 now includes an explicit fill-in-the-blank template:
   "Player [performs primary action] → earns [resource or reward] → uses it to
   [progress or unlock] → which unlocks [next challenge or content], repeating every
   [X seconds/minutes]."
   Plus two worked examples (match-3 garden; hex-grid troops). This template was
   directly used to articulate the core loop in this output. The resulting core loop
   description is more precise and operationally useful than the iteration-1 version.

6. PERFORMANCE TARGETS BENCHMARK CONTEXT [iteration-1 gap]
   STATUS: FULLY ADDRESSED — major improvement
   The skill v1.1.0 now includes a complete genre benchmark table in Section 1
   (Genre taxonomy) with D1 targets and CPI ranges per genre, PLUS a separate KPI
   gates table in Section 4 (Launch Status) with casual vs. mid-core targets for D1,
   D7, D30, CPI, D7 ROAS, and crash rate. This gave clear grounding for all targets
   in the output document. The output now shows benchmark-derived targets with
   explicit genre-midpoint rationale rather than assumed numbers without context.

7. BRAND VOICE SECTION DEPTH [iteration-1 gap]
   STATUS: NOT ADDRESSED
   The skill still requests "3–5 adjectives" without adding a "voice examples" field
   (taglines, example copy phrases). The brand voice section in this output remains
   adequate but thin — same as iteration 1. This remains a gap.

8. VERSION/CHANGELOG GUIDANCE [iteration-1 gap]
   STATUS: NOT ADDRESSED
   The skill still only captures "last updated" date with no changelog structure.
   Not blocking but still a valid gap for teams managing a living document.

=== NEW SKILL ELEMENTS IN ITERATION 2 (not in iteration-1 skill) ===

9. [NEEDS INPUT] DISCIPLINE
   The skill v1.1.0 now explicitly instructs: "mark unknown fields [NEEDS INPUT].
   Never invent numbers without flagging them as assumptions." This is an important
   addition. The output document now properly distinguishes between:
   - *(assumed)* — agent made a reasonable inference from available context
   - [NEEDS INPUT] — genuinely unknown; must come from user
   In iteration-1, the output used "Not yet tracked" inconsistently and sometimes
   left fields blank without signaling action required. The new discipline produces
   a more actionable document.

10. ATTRIBUTION TOOL AS EXPLICIT BLOCKER
    The skill v1.1.0 now notes attribution tool as "required before any paid UA"
    in BOTH Section 6 (Performance Targets) and Section 8 (UA Strategy). The Tips
    section also flags: "Attribution tool is a blocker — if the user has no MMP
    configured, flag it before UA discussions." This resulted in the output
    correctly calling out [NEEDS INPUT] for attribution in two places with explicit
    "hard blocker" language. Iteration-1 output was softer on this point.

11. IOS MEASUREMENT EXPLANATION
    The skill v1.1.0 now includes a specific note about ATT opt-in rates (~25–35%)
    and SKAdNetwork implications, plus a warning: "Don't promise iOS ROAS accuracy
    that doesn't exist post-ATT." This directly produced better iOS measurement
    guidance in the output, including the SKAN conversion value schema note.

=== IMPACT ON OUTPUT QUALITY ===

Compared to iteration-1 output (same game, same task):

BETTER in iteration 2:
- Core loop: More precisely articulated using the template; clearer action-reward-
  progression chain (puzzle battle → resources → construction → unlock → loop)
- Launch status: Now includes detailed soft launch market rationale explaining WHY
  CA/AU/NZ, not just listing them. This is more educationally valuable.
- KPI gates: Now properly derived from the genre benchmark table with mid-point
  rationale shown explicitly. Iteration-1 gates were reasonable guesses.
- UA channels: Now uses correct current names (Unity LevelPlay, Liftoff). Iteration-1
  output used outdated names (IronSource, Vungle).
- Attribution: Now flagged as a hard blocker in two sections.
- [NEEDS INPUT] fields: Now consistently used to distinguish unknowns from
  assumptions, producing a more actionable document.

SAME in iteration 2 (no change):
- Working title gap: Still no guidance; still had to invent a placeholder
- Competitive landscape: Still requires domain knowledge; no research scaffolding
- Brand voice: Still thin; no example copy phrases
- Proof points: Structurally identical; no new guidance for pre-launch state

=== OVERALL ASSESSMENT ===

The skill improved meaningfully in 5 of the 8 iteration-1 gaps. The improvements
that matter most — core loop template, genre benchmark table, soft launch rationale,
current UA naming, [NEEDS INPUT] discipline — all produced visible quality
differences in the output document. The output is more precise, more grounded in
benchmarks, and more actionable for a product team preparing for soft launch.

The three remaining gaps (working title guidance, competitive research scaffolding,
brand voice examples) are genuine but lower-priority refinements.

Rating: 9/10 (up from 8/10 in iteration 1)

The skill is now at production quality for a mobile game marketing context
document. The improvements in v1.1.0 directly address the most operationally
important gaps from the first iteration.
